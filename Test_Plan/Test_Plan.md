## 1. Introduction
This Test Plan describes the testing activities for the Kapital Bank website. The purpose of testing is to ensure that the website functions correctly and provides accurate information to users.

## 2. Test Objectives
- Verify that main website functionalities work as expected
- Ensure correct navigation between pages
- Identify UI and functional defects
- Validate content accuracy

## 3. Test Items
The following components will be tested:
- Homepage
- Cards pages
- Credits pages
- Deposits pages
- Online Banking page
- Contact page

## 4. Test Approach
Testing will be performed manually based on predefined test cases. Smoke testing will be executed before detailed testing. Regression testing will be performed after fixes.

## 5. Test Deliverables
- Test Plan
- Test Strategy
- Test Cases
- Bug Reports
- Test Summary Report

## 6. Roles and Responsibilities
- QA Engineer: Test planning, execution, and defect reporting

## 7. Test Schedule
- Test Planning: 1 day
- Test Case Preparation: 1 day
- Test Execution: 2 days
- Reporting: 1 day

## 8. Approval
The test results will be reviewed after completion of testing activities.
