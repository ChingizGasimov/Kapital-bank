## 1. Introduction
This Test Plan defines the testing activities for the Kapital Bank website to ensure correct feature behavior, usability, and content accuracy.

## 2. Test Objectives
- Validate key website features
- Ensure correct navigation and usability
- Detect functional and non-functional defects
- Verify content correctness

## 3. Test Items (Feature Coverage)

The following features will be tested:
- Website navigation and menu structure
- Product information presentation (cards, credits, deposits)
- Online banking entry point behavior
- Contact and branch information availability
- Search and navigation links
- Responsive layout behavior

## 4. Test Approach

Testing will include the following approaches:

- Functional testing of core features
- Exploratory testing to discover hidden defects
- Negative testing with invalid user actions
- UI and responsiveness testing on different screen sizes
- Content testing for accuracy and consistency
- Smoke testing on production environment

## 5. Non-Functional Testing (Minimum)

- Performance: basic page load observation
- Security: HTTPS usage and data visibility
- Accessibility: readability and usability checks

## 6. Test Deliverables
- Test Strategy
- Test Plan
- Test Cases
- Bug Reports
- Test Summary Report

## 7. Roles and Responsibilities
- QA Engineer: test planning, execution, reporting

## 8. Test Schedule
- Test planning: 1 day
- Test case preparation: 1 day
- Test execution: 2 days
- Reporting: 1 day

## 9. Test Environment
- Browsers: Chrome, Firefox
- OS: Windows 10
- Environment: Test and Production (limited)

## 10. Approval
Testing is considered complete after all planned activities are finished and results are reviewed.
